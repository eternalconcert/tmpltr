Features

Server

↑

TMPLTR comes with a bleeding edge node:http server. Starting is very easy. To set up a static file server, create a file called `server.mjs` with this content: 

    
    import { App } from '../src/app.mjs';

    const app = new App();

    app.serve();

That's it! Just place the files you want to server into a folder called `static` and start the server by typing: 

    
    node server.mjs

TMPLTR uses bleeding edge features of node. Threrefore you might need to add the --experimental-sqlite flag. You can also use the watch ability of node to make your live easier: 

    
    node --experimental-sqlite --watch server.mjs

Now the server will reload on every file change.

Template engine

As static page generator

↑

Although the template engine is designed to serve as an on-request function to generate pages in the server context, it can be used as a static page generator. To get familiar with the engine, this is the less complex approach. The main use case will be described afterwards.

Create a file called `page-generator.mjs` and paste the following code into that file: 

    
    import { App } from '../src/app.mjs';

    const app = new App();

    const pageContent = app.renderTemplate('static-page.html');

    console.log(pageContent);

Create a called `static-page.html` in a folder called `templates`, and plagce that folder next to the `page-generator.mjs`. Add the follwing text to `static-page.html`: 

    
    <h1>Pilot</h1>
    <p>Jack awakens disoriented in a jungle..</p>

Run the following command:

    
    node page-generator.mjs

This should print the following content to your screen:

<h1>Pilot</h1> <p>Jack awakens disoriented in a jungle..</p>

Okay, lame.. But at least it is rendered. Let's add some context! Change the content of `static-page.html` to match the following: 

    
    <h1>Pilot</h1>
    <h2>Episode: {{ episodeNumber }}</h2>
    <p>Jack awakens disoriented in a jungle..</p>

Modify the content of `page-generator.mjs`: 

    
    import { App } from '../src/app.mjs';

    const app = new App();

    const pageContent = app.renderTemplate('privacy.html', { episodeNumber: 1 });

    console.log(pageContent);

Run page-generator.mjs again, this should be the result:

<h1>Pilot</h1> <h2>Episode: 1</h2> <p>Jack awakens disoriented in a jungle..</p>

With these changes, we have added some context to our content. The episodeNumber can now be passed easily to the template.

ORM

Basic concepts

↑

TMPLTR comes with an ORM, that is inspired by the Django builtin ORM. The main idea is, to use class methods to retrieve objects from the database, and instance methods to modify this objects. A small example: 

    
      User.getAll();
      // returns a list of all users

      User.getByFilter({ lastName: 'Kwon' });
      // returns a list of all users with the last name 'Kwon'

      const user = User.getByFilter({ id: '3' })[0];
      // user contains now a single User-object.

      user.delete();
      // deletes the user from the database
